# 第一阶段细化计划：课堂助手 Web Demo 基础版

## Summary

第一阶段目标是从空仓库搭建一个可演示的高保真 Web Demo，完成“上传课堂/会议音频 → 本地 faster-whisper 转写 → 展示带时间戳字幕 → 预留实时理解辅助界面”的基础闭环。

计划文档保存路径：

`C:\Users\qinmy\Documents\WhisperProject\PHASE_1_PLAN.md`

## 技术选择

- 语音识别：本地 `faster-whisper`
- 运行环境：Windows 本机 GPU
- 后端：Python + FastAPI
- 前端：无构建静态 HTML + CSS + JavaScript
- 界面目标：高保真三栏 Demo
- 第一阶段输入：上传已录制音频，或粘贴 B 站/YouTube 视频链接；不做实时麦克风
- 第一阶段输出：转写文本、时间戳、模拟说话人、视频来源元数据、基础讨论占位信息

## 第一阶段要完成的工作

### 1. 项目初始化

- 创建前后端分离结构：
  - `frontend/`
  - `backend/`
  - `docs/`
  - `samples/`
- 创建项目说明：
  - `README.md`
  - `PHASE_1_PLAN.md`
- 配置 `.gitignore`，排除虚拟环境、模型缓存、上传音频、转写输出和前端构建产物。
- 写明本地运行方式和 GPU 依赖说明。

### 2. 后端服务

- 搭建 FastAPI 服务。
- 提供健康检查接口：
  - `GET /health`
- 提供音频上传转写接口：
  - `POST /api/transcribe`
- 提供视频链接转写接口：
  - `POST /api/transcribe-url`
- 接口接收音频文件：
  - `.mp3`
  - `.wav`
  - `.m4a`
  - `.mp4`
- 使用 `faster-whisper` 本地转写。
- 默认模型建议：
  - `small` 或 `medium`
- 默认运行配置：
  - `device="cuda"`
  - `compute_type="float16"`
- 如果 CUDA 不可用，返回清晰错误提示，不静默降级。
- 返回统一 JSON：

```json
{
  "language": "zh",
  "duration": 123.4,
  "segments": [
    {
      "id": 1,
      "start": 0.0,
      "end": 4.2,
      "speaker": "Speaker A",
      "text": "这里是转写内容。"
    }
  ]
}
```
- 视频链接输入通过 `backend/app/sources/` 的可插拔 resolver 架构处理，当前内置 B 站/YouTube，后续新增来源只需要注册新的 resolver。

### 3. 前端高保真界面

- 使用静态 HTML + CSS + JavaScript 搭建界面，不依赖 npm。
- 实现三栏布局：

```text
左侧：音频上传区 + 转写字幕流
中间：当前讨论焦点 / 主题摘要 / 争论点占位
右侧：AI 辅助按钮 / 使用说明 / 状态卡片
```

- 左侧功能：
  - 上传音频
  - 粘贴 B 站或 YouTube 链接
  - 显示上传状态
  - 显示转写进度
  - 显示带时间戳字幕
  - 显示 `Speaker A/B`
- 中间功能：
  - 显示“当前讨论焦点”
  - 第一阶段可用规则或占位摘要生成：
    - 从前几段文本合并生成简短主题
    - 不接入 LLM
- 右侧功能：
  - 展示但暂不完全实现这些按钮：
    - “我没听懂”
    - “刚刚谁反对谁？”
    - “帮我生成一个问题”
    - “我离开了 3 分钟，补一下”
    - “生成会后行动项”
  - 点击未实现按钮时显示“第二阶段实现”的提示。

### 4. 用户体验细节

- 高保真但不过度复杂，重点体现课堂助手产品感。
- 使用清晰的状态：
  - 未上传
  - 上传中
  - 转写中
  - 转写完成
  - 转写失败
- 转写失败时显示可理解错误。
- 字幕列表支持滚动。
- 时间戳格式统一为 `mm:ss`。
- 长文本自动换行。
- 空状态要有引导文案。

### 5. 第一阶段不做的内容

- 不做实时麦克风输入。
- 不做真正说话人分离，只使用模拟 `Speaker A/B`。
- 不接入 LLM。
- 不做“我没听懂”的真实解释逻辑。
- 不做用户登录。
- 不做数据库。
- 不做云端部署。
- 不保存永久会议记录。

## Test Plan

- `GET /health` 返回正常状态。
- 上传合法音频后，后端返回 `segments`。
- 上传不支持格式时，后端返回清晰错误。
- CUDA 不可用时，后端返回明确 GPU 配置错误。
- 前端能完成上传并展示字幕。
- 前端能展示转写中、完成、失败三种状态。
- 字幕时间戳格式正确。
- 三栏界面在常见桌面浏览器宽度下可用。
- 未实现按钮不会报错，只显示第二阶段提示。

## Acceptance Criteria

第一阶段完成时，用户可以在本地启动前端和后端，上传一段中文课堂或会议录音，并在高保真三栏界面中看到转写字幕、时间戳、模拟说话人和课堂助手式功能入口。

这版 Demo 不需要真正实时，但必须让人一眼看出后续会发展成“实时理解无障碍助手”，而不是普通会议纪要工具。
